# Getting help

This project is open-source software with community-and-paid support. Use the right channel for the right question.

## Free / community support

| What | Where | When to use |
|---|---|---|
| Usage questions, deployment help, "how do I…" | [GitHub Discussions](https://github.com/pandora-cloud/govwin-hubspot-integration/discussions) | Default starting point for any question that isn't a confirmed bug or a security issue. |
| Confirmed bugs | [GitHub Issues](https://github.com/pandora-cloud/govwin-hubspot-integration/issues) | You can reproduce a defined defect. Include redacted logs and steps. |
| Feature requests | [GitHub Issues](https://github.com/pandora-cloud/govwin-hubspot-integration/issues) | The feature serves federal AWS partners running this pipeline. |
| Security disclosures | [Private security advisory](https://github.com/pandora-cloud/govwin-hubspot-integration/security/advisories/new) or `pc@pandoracloud.net` | Credentials, vulnerabilities, ACE submission integrity, supply-chain concerns. **Never** open a public issue. See [SECURITY.md](SECURITY.md). |

Response time for community support is best-effort. Expectations:

- Discussions: best-effort, no SLA.
- Issues: 5 business days for first response.
- Security: 7 calendar days for first response, 30 days for fix or mitigation.

## Paid support

If you need any of the following, contact <pc@pandoracloud.net>:

- Deployment assistance into your AWS account.
- Federal compliance review (NIST 800-53 / CMMC L2 / FedRAMP Moderate-aligned).
- Prioritized feature work or roadmap acceleration.
- Production incident support with a guaranteed response window.
- Customizations that fork from this repo (a separate engagement; we don't merge customer-specific code into the public codebase).

Pandora Cloud is a Woman-Owned Small Business (WOSB), an AWS Advanced Tier Partner, and a federal contractor. Engagements are typically scoped against our GSA MAS Schedule (`47QTCA24D00D3`) or a direct services contract.

## What this project is not

We will not provide free support for:

- Forks that have been substantially modified, where the modifications are the source of the bug.
- Commercial integration scenarios where the support effort exceeds what an OSS project can sustain.
- Adapters for non-supported CRMs (Salesforce, Pipedrive, Dynamics, etc.). Those are out of scope per [CONTRIBUTING.md](CONTRIBUTING.md).
